4th Quarter: JavaScript


Lesson 1: Introduction to JavaScript

What is a Script? A script is a set of computer instructions or commands that connects components to accomplish a task.

JavaScript:

  • A powerful client-side scripting language interpreted by web browsers.
  • Created by Brendan Eich.
  • Enables developers to build interactive and functional websites.
  • Originally called Mocha → LiveScript → JavaScript.
  • Not related to Java and is case-sensitive.
  • Normally embedded in HTML files (*.htm or *.html).

Characteristics of JavaScript:

  • Interpreted Language: Runs directly in the browser without compilation.
  • Scripted Language: Designed for simple or repetitive tasks.
  • Client-Side Scripting: Executes on the user's machine (not the server).
  • Object-Oriented Programming (OOP): Treats elements as objects.

Important Terms:

  • Objects: Items that exist in the browser.
  • Methods: Actions performed on objects.
  • Properties: Attributes or subsections of objects.
  • Events: Actions like page load, clicks, or mouseovers.
  • Parameters: Data describing an object’s characteristics.
  • Type: Variable type, e.g., integer, string, float, Boolean.
  • Type Conversion: Changing a variable from one type to another.

Summary: JavaScript is a client-side scripting language used to create interactive, functional websites. It runs in browsers, supports object-oriented programming, and works with events, methods, and properties to control web page behavior.

Lesson 2: Placing Scripts & JavaScript Guidelines

Where can you place Scripts? You can control when your JavaScript code runs by placing it in different parts of the HTML file.

  • HEAD: Scripts here are pre-loaded, executed before any events are triggered. Ideal for function calls.
  • BODY: Scripts here run when the page loads.
  • HEAD & BODY: You can place multiple scripts in both sections as needed.
  • EXTERNAL: Scripts saved in .js files can be linked using <script src="myJavaScript.js"></script>. This is helpful for using the same code across multiple HTML files.

Comments in JavaScript:

  • Single-line comment: // This is a comment
  • Multi-line comment: /* This is a comment */

JavaScript Guidelines:

  1. Case Sensitive: Variable and function names must match exactly. myHome ≠ MyHome
  2. White Space: Spaces are ignored, but using them improves readability. Example: var name = "Grace";
  3. Breaking up code lines: You can break lines with a backslash (\), but do not split commands improperly. Example: document.write("wow\\Philippines");

Summary: Scripts can be placed in the HEAD, BODY, or externally. Always follow JavaScript rules for case sensitivity, proper spacing, and line breaks to avoid errors.

Lesson 3: JavaScript Events & Actions

What are JavaScript Events? Events are actions or occurrences that happen in the browser, such as:

  • Loading a page (onload)
  • Clicking a button (onclick)
  • Hovering over an element (onmouseover)
  • Typing in a form (oninput)

How to Handle Events:

  • Assign a function to an HTML element’s event attribute: <button onclick="greet()">Click Me</button>
  • Write the function in a <script> tag: function greet() { alert("Hello!"); }
  • Functions can be placed in HEAD, BODY, or in external JS files.

Common JavaScript Actions:

  • alert(): Shows a popup message.
  • document.write(): Writes text to the HTML document.
  • console.log(): Displays messages in the browser console (useful for debugging).
  • Changing content or styles: JavaScript can modify text, colors, images, or any HTML element dynamically.

Summary: Events allow JavaScript to respond to user actions or browser activity. By combining functions with events, we can create interactive and dynamic web pages.
